2. HIGH: SSRF via Notification Channel Webhook URLs (No

File: app/api/notification-channels/route.ts:136-182

Mechanism: The webhook URL validation in the notification channel creation flow uses validateWebhookUrl() which calls isPrivateOrLocalhost() to check the hostname string against a blocklist and regex patterns. It does NOT resolve DNS and check the resolved IPs. Compare this with
the monitor creation flow which uses assertSafeUrl() frh resolves DNS via dns.lookup() and checks all resolvedIPs against private ranges.

The notification channel SSRF check (isPrivateOrLocalhost) only blocks literal hostnames like localhost, 127.0.0.1, 169.254.169.254, and private IP regex patterns. An attacker who controls a DNS A record can point any domain to an internal IP.

Impact: An authenticated user (with DEVELOPER+ role) can create a webhook notification channel with a URL like
https://attacker-controlled.com/webhook where that doma2.30.0.1 (the Docker bridge). When a notification fires, the server will make an HTTP request to the internal network.
